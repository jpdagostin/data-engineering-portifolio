Projeto de Criptografia com Chaves Públicas e Privadas

Descrição do Projeto:

Este projeto foi desenvolvido como parte de um trabalho de faculdade, onde explorei os conceitos fundamentais de criptografia utilizando a geração de chaves públicas e privadas diretamente pelo terminal CMD, sem o uso de bibliotecas ou linguagens de programação. A proposta foi criptografar e descriptografar mensagens enviadas pelo professor utilizando comandos de linha de comando.

Tecnologias e Ferramentas Utilizadas:

  • Terminal CMD (Comandos de linha de comando)
  • OpenSSL (Ferramenta de criptografia para geração de chaves e criptografia)

Funcionamento e Metodologia:

O projeto utiliza criptografia assimétrica, onde são geradas chaves públicas e privadas por meio de comandos no terminal. A chave pública foi usada para criptografar a mensagem, enquanto a chave privada foi usada para descriptografá-la. As etapas principais foram:

  1. Geração das Chaves: No terminal CMD, utilizei o OpenSSL para gerar um par de chaves (pública e privada). A chave pública foi compartilhada com o professor, que enviou uma mensagem criptografada.
  2. Criptografia: O professor utilizou minha chave pública para criptografar as mensagens, tornando-as ilegíveis para qualquer pessoa que não possuísse minha chave privada.
  3. Descriptografia: Com a minha chave privada, consegui descriptografar as mensagens, restaurando-as ao formato de texto legível.

Comandos Utilizados para Geração das Chaves e Criptografia:

  • Navegar até o diretório correto: 
    
cd "C:/Users/dagos/OneDrive/Desktop"
  • Gerar a chave privada: 
    
openssl genpkey -algorithm RSA:2048 -out jpd_private_key.key
  • Gerar a chave pública correspondente: 
    
openssl rsa -in jpd_private_key.key -pubout -out jpd_public_key.key
  • Gerar um certificado de solicitação: 
    
openssl req -new -key jpd_private_key.key -out certificado_jpd.csr
  • Gerar o hash de um arquivo usando SHA-512: 
    
openssl dgst -sha512 -binary arquivoName_RA_Frase.txt > arquivoHash.bin
  • Assinar o hash com a chave privada: 
    
openssl pkeyutl -sign -in arquivoHash.bin -inkey jpd_private_key.key -out hashAssinadoJP.bin
  • Criptografar a mensagem com a chave pública do professor: 
    
openssl pkeyutl -encrypt -pubin -inkey chave_publica_Mozart.pem 
-in arquivoName_RA_Frase.txt -out arquivoCryptJP.bin

Desafios Enfrentados:

Um dos principais desafios foi garantir que as chaves geradas fossem seguras e que os processos de criptografia e descriptografia funcionassem corretamente com os diferentes arquivos de mensagem. Também foi necessário assegurar que a troca de chaves públicas entre eu e o professor fosse feita de maneira correta para garantir a privacidade das mensagens.

Resultados:

O projeto foi um sucesso, permitindo que as mensagens criptografadas fossem enviadas com segurança utilizando a chave pública e descriptografadas com a chave privada no terminal CMD. Essa experiência permitiu um melhor entendimento dos conceitos de criptografia assimétrica e segurança da informação.
